pickled-plums

Real-time telemetry, pickled for precision

The Problem it Solves

The issue was raised while we were setting up a pipeline for Azure IoT Hub. The way IoT Hub metered billing based on data throughput and volume was really aggressive. Therefore, pickled-plums is an attempt to cut down on billing while maintaining minimal, custom telemetry using self-hosted instances.

  • Reduces telemetry costs significantly compared to cloud-based solutions
  • Enables real-time data processing on edge devices
  • Customizable for IoT and edge computing use cases
  • Runs efficiently on bare-minimum hardware

Challenges I Ran Into

Building the low-level design of the client software was a major hurdle. Since we wanted to fit it into a bare-minimum edge machine, the more we refined it, the more minimal it felt it could be.

I overcame this by iteratively optimizing the codebase, stripping it down to essentials while ensuring functionality. Rapid prototyping and testing on edge hardware helped refine the solution.

Tech Stack

  • Python
  • Golang
  • Grafana

Setting up on VPS

Self-Hosted Telemetry Service

pip3 install -r requirements.txt

Run Simulation

make simulation

Test API Endpoint

make curl

Run Individual Processes

make process p=p1
make process p=p2
make process p=p3
make process p=p4

Each process will stream data to:

output/csv/.csv
